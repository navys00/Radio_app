# Продуктовые политики: «Радио 1941–1945»

Зафиксированные решения для согласованного поведения приложения и приёмки.

## 1. Нет треков на выбранный год на станции

- Воспроизводится **статический шум** (тот же сценарий, что и на «пустом» эфире), в метаданных плеера указываются город, частота и год.
- **Нет** автоматического перехода на другой год или станцию (предсказуемость настройки).

## 2. Кнопка Stop

- **Частота и год** в UI и в `AsyncStorage` **сохраняются**.
- Очередь треков **сбрасывается** (`reset`), внутренний ключ текущего эфира сбрасывается, чтобы следующий запуск или смена года снова загрузили контент.
- Повторное **Play** после Stop снова «настраивает» эфир на текущей частоте и годе (короткий шум переключения по общим правилам).

## 3. «Пустые» частоты (между станциями)

- Если частота **не** захватывает станцию, но попадает в слот из `offStationSlots` в [stations.json](../src/assets/stations.json), воспроизводится контент слота для выбранного года (или шум, если в слоте нет треков на этот год).
- Переключение между слотами стабилизируется **гистерезисом**; детали — [tz-off-station-slots.md](tz-off-station-slots.md).
- Если нет ни станции, ни слота — **шум**.

## 4. Web (Expo Web)

- Целевая платформа релиза: **только нативные Android и iOS**.
- На **web** нативное воспроизведение через `react-native-track-player` **не** включается; UI шкалы и ручек может работать, но аудио на web **вне области** текущего MVP (без ошибок в консоли от плеера).

## 5. Shuffle и фильтрация по жанру

- По умолчанию **выключено**; отдельный флаг в настройках не входит в текущую поставку (можно добавить позже).

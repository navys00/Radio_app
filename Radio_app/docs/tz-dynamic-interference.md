# ТЗ (эпик V3): динамические помехи при настройке

Документ задаёт требования к будущей реализации; в текущей сборке реализована только фиксированная вставка шума при смене источника (см. `SWITCH_NOISE_MS` в [player.ts](src/services/player.ts)).

## 1. Цель

Имитировать качество приёма средневолнового приёмника: при отстройке от номинала станции или при быстром вращении ручки растут шипение, подтон, «залипание» соседнего канала.

## 2. Входные параметры (модель)

- `deltaKhz` — расстояние по частоте до номинала ближайшей «захваченной» станции (или до центра слота вне станций).
- `tuningVelocity` — скорость изменения частоты (кГц/с), сглаженная за 100–200 мс.
- `signalQuality01` — абстрактный коэффициент 0…1 (опционально от «лампы»/режима приёмника).

## 3. Аудио-архитектура

- **Слой A:** декодированный полезный сигнал (очередь треков).
- **Слой B:** шумовая подложка (один или несколько зацикленных буферов: белый/розовый шум, импульсы).
- **Микшер:** `output = A * gA + B * gB`, где `gB` растёт с `|deltaKhz|` и `tuningVelocity`, `gA` убывает; суммарная громкость ограничена (например ≤ 1.0 после лимитера), пиковый уровень не выше заданного LUFS относительно сигнала.

## 4. Реализация по платформам (ориентир)

- Предпочтительно: нативный микшер / двойной `TrackPlayer` (если поддерживается) или один поток с **Web Audio–подобной** цепочкой на нативном DSP.
- Упрощённый вариант: 3–5 заранее сведённых клипов «низкий / средний / высокий шум» и кроссфейд по дискретным ступеням без DSP.

## 5. Безопасность и UX

- Плавное нарастание/спад помехи (10–150 мс), без щелчков.
- В настройках: «Интенсивность помех» (выкл / низко / норма / высоко) и предпрослушка не обязательны в первой итерации.

## 6. Приёмка (когда эпик в работе)

- При медленной настройке вблизи станции помеха меняется монотонно и предсказуемо.
- При резком пролёте диапазона нет клипов и артефактов на устройствах mid-range (Android 12+, iOS 16+).

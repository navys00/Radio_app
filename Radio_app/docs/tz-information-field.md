# ТЗ: информационное поле (шкала и станции)

Документ фиксирует требования к блоку отображения частот и станций. Источник констант в коде: [`src/data/radioData.ts`](src/data/radioData.ts) (`INFORMATION_FIELD_BANDS`, `stationsInInformationBand`), реализация UI: [`src/components/FrequencyScale.tsx`](src/components/FrequencyScale.tsx).

## 1. Состав информационного поля

В поле входят **только** элементы, относящиеся к шкале и подписям:

- легенда («СРЕДНИЕ ВОЛНЫ · кГц»);
- строки названий станций по полосам частот;
- окно градуировки (засечки, маркеры станций);
- строки цифровых отметок кГц по полосам.

**Не входят** в информационное поле: ламповый индикатор, ручки настройки, стрелка-указатель (стрелка визуально накладывается на поле, но логически позиционируется отдельно — см. п. 5).

На всех размерах экрана используется **один** компонент шкалы; отдельная вёрстка только для планшета в этой версии ТЗ **не** требуется.

## 2. Полосы по диапазонам частот (by band)

Используются **три** непересекающихся диапазона (кГц, границы **включительно** для станций):

| Полоса | ID | Диапазон станций (кГц) | Подпись в ТЗ |
|--------|-----|-------------------------|--------------|
| 1 | `mw_low` | 520–879 | Нижняя треть MW |
| 2 | `mw_mid` | 880–1239 | Средняя треть MW |
| 3 | `mw_high` | 1240–1600 | Верхняя треть MW |

**Правило границы:** станция с частотой **F** относится к полосе, где `minKhz ≤ F ≤ maxKhz` для соответствующей записи в `INFORMATION_FIELD_BANDS`. Диапазоны не пересекаются: **880** относится ко второй полосе, **879** — к первой; **1240** — к третьей.

**Отбор станций в строке:** в строке полосы показываются **только** станции, попадающие в её диапазон; порядок — **строго по возрастанию** частоты.

## 3. Строки внутри полосы

- На каждую полосу — **одна** горизонтальная строка названий (несколько станций по шкале, позиция по частоте).
- **Соединительная линия** от подписи города к зоне шкалы (`hairline`) — **желательно** (как на шкалах 1930-х), остаётся в реализации.

## 4. Подписи кГц по полосам

- Каждая полоса имеет **свою** строку цифр частот; набор значений задаётся полем `axisLabelsKhz` у соответствующей записи в `INFORMATION_FIELD_BANDS` (согласовано с визуальной плотностью).
- Единицы измерения: **один раз** в легенде («кГц»), у отдельных цифр суффикс **не** дублируется.
- Шаг подписей внутри полосы — **кратен практичности чтения** (80–100 кГц в средней полосе и т. п.), без обязательных «50 кГц» на узком экране.

## 5. Стрелка и геометрия

- Стрелка **визуально** совмещается с окном градуировки (движется вдоль той же логической шкалы 520–1600 кГц).
- Вертикальное выравнивание: **без «магического» фиксированного `paddingBottom` в экране** как единственного источника правды. Компонент шкалы передаёт вверх по колбэку **`onDialGeometry`**:
  - `trackWidth` — ширина окна градуировки (для расчёта положения стрелки по X);
  - `pointerLiftFromBottom` — расстояние от нижнего края корневого контейнера шкалы до нижнего края окна градуировки (для отступа слоя стрелки снизу).

## 6. Визуальный стиль (1930-е)

- Палитра задаётся **качественно** (кремовый циферблат, тёмная рамка, бронзовая кайма окна шкалы); точные HEX могут подстраиваться в коде без пересмотра ТЗ.
- **Serif** для подписей и легенды, капитель для легенды, **рамка окна шкалы** — обязательные элементы образа «гражданский приёмник 1930-х».

## 7. Узкий экран и доступность

- При ширине **≥ 320 dp**: допускаются **уменьшение шрифта** и **перенос названия города до 2 строк** (`numberOfLines`, `adjustsFontSizeToFit`); **скрытие станций** с шкалы не допускается — все станции из конфигурации должны оставаться представленными в соответствующих полосах.
- **TalkBack:** для ручек частоты и громкости задаются `accessibilityLabel`; расширенное зачитывание частоты/станции по шкале в этой версии ТЗ **не** обязательно (можно добавить отдельным пунктом позже).

## 8. Приёмка

- На ширине **≥ 360 dp** не допускается визуальное **перекрытие** двух соседних подписей **городов** в одной строке полосы при типовом наборе из 16 станций (контроль глазами + скриншоты).
- Все станции из `stations.json` **видны** в одной из трёх строк полос (ни одна не выпадает из диапазонов 520–1600).

## Связь с настройкой приёмника

Глобальный диапазон настройки `FREQUENCY_RANGE` (520–1600 кГц) не меняется; полосы информационного поля служат только для **группировки подписей** на UI и **не** ограничивают алгоритм `resolveStationForFrequency`.

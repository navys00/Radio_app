# Радио ЭФИР 1941-1945 (Expo CLI MVP)

MVP Android-приложения с офлайн-аудио и фоновым воспроизведением через `react-native-track-player`. Интерфейс имитирует ламповый приёмник 1940-х: фон корпуса, шкала частот (графика), стрелка-указатель, ламповый индикатор и **единственное управление на экране — ручка настройки** (жест вверх/вниз по ручке для плавной настройки с привязкой к дискретным частотам).

## Запуск разработки

1. Установить зависимости:

```bash
npm install
```

2. Сгенерировать Android нативную часть (первый раз или после изменения `app.json`):

```bash
npm run prebuild:android
```

3. Запустить development build:

```bash
npm run android
```

Важно: проект использует `react-native-track-player`, поэтому требуется Development Build, а не Expo Go.

## Контент и структура данных

- Конфигурация станций: `src/assets/stations.json`
- Маппинг и валидация аудио-id: `src/assets/audioMap.ts`
- Все `file` в `stations.json` должны соответствовать именам ресурсов в `android/app/src/main/res/raw`.
- Белый шум: ресурс `static` (`static.mp3`).

## Что реализовано в MVP

- Дискретные частоты (станции + «пустые» частоты с белым шумом); на экране нет слайдера, селектора года и кнопок Play/Pause/Stop.
- Очередь станции объединяет **все** треки из `stations.json` по годам (1941→1945) в один цикл.
- Переключение частоты с коротким шумом (~800 ms), затем новая станция или шум.
- Эффект «прогрева» при старте (затемнение и плавное проявление).
- Подсказка «Вращайте ручку настройки» скрывается после первого жеста на ручке.
- Фоновое воспроизведение с remote-кнопками (Play/Pause/Stop) **только в уведомлении**.
- Сохранение и восстановление **только частоты** через AsyncStorage.

Внешний вид приёмника собран **вёрсткой** (Views, стили, подписи шкалы из данных станций), без PNG-ассетов интерфейса.

## Release-сборка

- Internal dev build: `npm run build:android:dev`
- Internal preview APK/AAB: `npm run build:android:preview`
- Production AAB: `npm run build:android:release`
